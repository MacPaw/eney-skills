This extension demonstrates two powerful patterns:

1. **Reading macOS system data** with [`useSQL`](/docs/hooks/use-sql) — querying the Apple Notes SQLite database
2. **Writing to macOS apps** with [`useAppleScript`](/docs/hooks/use-apple-script) — appending content through AppleScript

---

## What it does

The widget lets the user select a note from a dropdown (populated from the Notes database) and append rich text content to it using a rich text editor. Content is converted from Markdown to HTML before being written to Notes.

---

## Reading notes with useSQL

The `helpers/use-notes.ts` file queries the Apple Notes SQLite database directly:

```typescript
import { homedir } from "os";
import { resolve } from "path";
import { useSQL, useLogger } from "@eney/api";

const NOTES_DB = resolve(
  homedir(),
  "Library/Group Containers/group.com.apple.notes/NoteStore.sqlite",
);

const query = `
  SELECT
    note.ztitle1 AS title,
    note.z_pk AS pk,
    folder.ztitle2 AS folder
  FROM ziccloudsyncingobject AS note
  INNER JOIN ziccloudsyncingobject AS folder ON note.zfolder = folder.z_pk
  WHERE note.ztitle1 IS NOT NULL AND note.zmarkedfordeletion != 1
  ORDER BY note.zmodificationdate1 DESC
`;

export const useNotes = () => {
  const logger = useLogger();
  const { data, isLoading } = useSQL<{ title: string; pk: number; folder: string }[]>(
    NOTES_DB,
    query,
  );

  logger.debug(`Fetched ${data?.length ?? 0} notes`);

  return { data: data ?? [], isLoading };
};
```

<Callout type="info">
  `useSQL` is a React hook — call it inside your widget component or a custom hook, not at the module level.
</Callout>

## Writing notes with useAppleScript

`useAppleScript` is called at the top of the component like any other hook, and `runScript` is used inside the async `appendToNote` helper defined within the component:

```typescript
import { useAppleScript } from "@eney/api";
import markdownit from "markdown-it";
import sanitizeHtml from "sanitize-html";

function AppendToNote(props: Props) {
  const runScript = useAppleScript();

  async function appendToNote(noteName: string, content: string) {
    // Convert Markdown to sanitized HTML before writing to Notes
    const html = sanitizeHtml(markdownit({ breaks: true }).render(content));
    const escaped = escapeDoubleQuotes(html);

    return runScript(`
      tell application "Notes"
        set targetNote to first note whose name is "${escapeDoubleQuotes(noteName)}"
        set body of targetNote to (body of targetNote) & "${escaped}"
      end tell
    `);
  }

  // ...
}
```

<Callout type="warn">
  Always escape user-provided strings before interpolating them into AppleScript. Use a helper to replace `\` and `"` characters.
</Callout>

## Widget component

```tsx
import { useEffect, useState } from "react";
import {
  Action,
  ActionPanel,
  defineWidget,
  Form,
  Paper,
  CardHeader,
  useCloseWidget,
  useAppleScript,
} from "@eney/api";
import { useNotes } from "../helpers/use-notes.js";

const NEW_NOTE_VALUE = "__new_note__";

function escapeDoubleQuotes(value: string) {
  return value.replace(/\\/g, "\\\\").replace(/"/g, '\\"');
}

function AppendToNote(props: Props) {
  const closeWidget = useCloseWidget();
  const runScript = useAppleScript();
  const { data: notes, isLoading: isLoadingNotes } = useNotes();

  const [noteName, setNoteName] = useState(props.noteName ?? NEW_NOTE_VALUE);
  const [content, setContent] = useState(props.content ?? "");
  const [isAppending, setIsAppending] = useState(false);

  async function appendToNote(noteName: string, content: string) {
    const html = sanitizeHtml(markdownit({ breaks: true }).render(content));
    const escaped = escapeDoubleQuotes(html);

    if (noteName === NEW_NOTE_VALUE) {
      return runScript(`
        tell application "Notes"
          set newNote to make new note
          set body of newNote to "${escaped}"
        end tell
      `);
    }

    return runScript(`
      tell application "Notes"
        set targetNote to first note whose name is "${escapeDoubleQuotes(noteName)}"
        set body of targetNote to (body of targetNote) & "${escaped}"
      end tell
    `);
  }

  async function onSubmit() {
    if (!content.trim()) return;
    setIsAppending(true);
    let finalContext = "";
    try {
      await appendToNote(noteName, content);
      const noteDisplay = noteName === NEW_NOTE_VALUE ? "a new note" : `the note "${noteName}"`;
      finalContext = `Content appended successfully to ${noteDisplay}`;
    } catch (error) {
      finalContext = error instanceof Error ? error.message : "Failed to append to note";
    } finally {
      setIsAppending(false);
      closeWidget(finalContext);
    }
  }

  return (
    <Form
      header={<CardHeader title="Notes" iconBundleId="com.apple.Notes" />}
      actions={
        <ActionPanel>
          <Action.SubmitForm
            title={isAppending ? "Appending..." : "Append to Note"}
            onSubmit={onSubmit}
            style="primary"
            isDisabled={!content.trim()}
            isLoading={isAppending}
          />
        </ActionPanel>
      }
    >
      <Form.Dropdown name="noteName" value={noteName} onChange={setNoteName}>
        <Form.Dropdown.Item title="New Note" value={NEW_NOTE_VALUE} />
        {notes.allNotes.map((note) => (
          <Form.Dropdown.Item key={note.id} title={note.title} value={note.title} />
        ))}
      </Form.Dropdown>
      <Form.RichTextEditor value={content} onChange={setContent} isInitiallyFocused />
    </Form>
  );
}
```

## Key patterns to copy

- Call `useAppleScript()` at the top of the component like any other hook — use the returned `runScript` function inside async helpers defined within the component
- Convert Markdown to sanitized HTML before writing to Notes — the Notes app stores HTML, not plain text
- Use `useSQL` + a separate helper file to keep database logic out of components
- Use `useLogger` inside hooks for debug output that doesn't pollute the UI
- Always call `closeWidget(context)` with a meaningful context string so Eney knows what happened
