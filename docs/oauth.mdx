Many extensions need to access third-party APIs that require user authorization — Spotify, Google, Dropbox, and others. The `@eney/api` package provides a [`useOAuth`](/docs/hooks/use-oauth) hook that handles the entire PKCE sign-in flow. You control the UI for every state — loading, error, and authenticated.

## How it works

1. The [`useOAuth`](/docs/hooks/use-oauth) hook checks for `OAUTH_ACCESS_TOKEN` and `OAUTH_REFRESH_TOKEN` environment variables.
2. If present, tokens are returned immediately.
3. If not, a local server starts on port `1798` and the provider's authorization page opens in the browser.
4. After the user authorizes, the hook exchanges the authorization code for tokens using PKCE (S256).
5. Tokens are sent to Eney via MCP notification so they persist across sessions — meaning the user only signs in once.

<Callout type="info">
  PKCE (Proof Key for Code Exchange) is designed for public clients that cannot securely store a client secret. Your extension does not need a client secret — only a client ID.
</Callout>

## Setup

<Steps>

<Step>
### Register an OAuth application with your provider

Create an OAuth app in your provider's developer console. Set the redirect URI to:

```
http://127.0.0.1:1798/callback
```

Copy the **client ID** — you will not need a client secret.

Make sure the provider supports PKCE with the S256 code challenge method. Most major providers do, including Google, Spotify, GitLab, Dropbox, and Zoom.
</Step>

<Step>
### Call useOAuth in your component

Define your OAuth config and pass it to the `useOAuth` hook. The hook returns the current auth state — render your UI accordingly:

```tsx
import { z } from "zod";
import { defineWidget, useOAuth, Paper, ActionPanel, Action } from "@eney/api";
import type { OAuthConfig } from "@eney/api";

const schema = z.object({});

const spotifyOAuth: OAuthConfig = {
  clientId: "your-spotify-client-id",
  authorizeUrl: "https://accounts.spotify.com/authorize",
  tokenUrl: "https://accounts.spotify.com/api/token",
  scopes: ["user-top-read"],
};

function TopTracks() {
  const { isLoading, error, data, retry } = useOAuth(spotifyOAuth);

  if (isLoading) return <Paper markdown="Signing in..." />;

  if (error) {
    return (
      <Paper
        markdown={`**Sign in failed**\n\n${error}`}
        actions={<ActionPanel><Action title="Retry" onAction={retry} style="primary" /></ActionPanel>}
      />
    );
  }

  // data.accessToken is available here
  return <Paper markdown="Authenticated!" />;
}

const TopTracksWidget = defineWidget({
  name: "spotify-top-tracks",
  description: "Show your top Spotify tracks",
  schema,
  component: TopTracks,
});

export default TopTracksWidget;
```
</Step>

</Steps>

## Configuration reference

The `OAuthConfig` object passed to `useOAuth` accepts the following options:

| Property       | Type       | Default       | Required | Description                                         |
| -------------- | ---------- | ------------- | -------- | --------------------------------------------------- |
| `clientId`     | `string`   | —             | Yes      | The OAuth client ID from your provider              |
| `authorizeUrl` | `string`   | —             | Yes      | The provider's authorization endpoint               |
| `tokenUrl`     | `string`   | —             | Yes      | The provider's token exchange endpoint              |
| `scopes`       | `string[]` | —             | Yes      | The OAuth scopes to request                         |
| `callbackPort` | `number`   | `1798`        | No       | Port for the local callback server                  |
| `callbackPath` | `string`   | `"/callback"` | No       | Path for the OAuth redirect                         |

## Environment variables

On the first run, the full browser-based OAuth flow is triggered. After tokens are obtained, Eney receives them via MCP notification and provides them as environment variables on subsequent sessions:

| Variable              | Description                   |
| --------------------- | ----------------------------- |
| `OAUTH_ACCESS_TOKEN`  | The OAuth access token        |
| `OAUTH_REFRESH_TOKEN` | The refresh token, if present |

When these variables are set, the hook skips the browser flow entirely and returns the cached tokens immediately.

## Manifest configuration

To allow Eney to inject the OAuth tokens, add placeholders in your `manifest.json` server environment:

```json
{
  "server": {
    "mcp_config": {
      "command": "node",
      "args": ["${__dirname}/index.js"],
      "env": {
        "OAUTH_ACCESS_TOKEN": "${OAUTH_ACCESS_TOKEN}",
        "OAUTH_REFRESH_TOKEN": "${OAUTH_REFRESH_TOKEN}"
      }
    }
  }
}
```

<Callout type="warn">
  The local callback server binds to `127.0.0.1` only and shuts down after receiving the callback. If the user does not complete the authorization within 60 seconds, the flow times out and the hook sets the `error` state.
</Callout>

## Token refresh

Access tokens expire. The `useOAuth` hook returns a `refresh()` function that exchanges the refresh token for a new access token:

```tsx
const { data, refresh } = useOAuth(oauthConfig);
const [token, setToken] = useState<string | null>(null);

useEffect(() => {
  if (data) setToken(data.accessToken);
}, [data]);

async function fetchWithRefresh(url: string) {
  let res = await fetch(url, {
    headers: { Authorization: `Bearer ${token}` },
  });

  if (res.status === 401) {
    const newTokens = await refresh();
    setToken(newTokens.accessToken);
    res = await fetch(url, {
      headers: { Authorization: `Bearer ${newTokens.accessToken}` },
    });
  }

  return res;
}
```

When `refresh()` is called, the hook:
1. POSTs to the provider's `tokenUrl` with `grant_type=refresh_token`
2. Returns the new `{ accessToken, refreshToken }`
3. Sends updated tokens to Eney so they persist across sessions

<Callout type="info">
  Not all providers issue refresh tokens by default. Some require specific scopes — for example, Google needs `access_type=offline` and Spotify includes refresh tokens automatically. If no refresh token was issued, calling `refresh()` will throw an error.
</Callout>

## Full example

A complete widget that shows a user's top Spotify tracks with token refresh:

```tsx
// components/spotify-top-tracks.tsx
import { useState, useEffect } from "react";
import { z } from "zod";
import {
  defineWidget,
  useOAuth,
  useCloseWidget,
  Paper,
  ActionPanel,
  Action,
} from "@eney/api";
import type { OAuthConfig } from "@eney/api";

const schema = z.object({});

const spotifyOAuth: OAuthConfig = {
  clientId: "your-spotify-client-id",
  authorizeUrl: "https://accounts.spotify.com/authorize",
  tokenUrl: "https://accounts.spotify.com/api/token",
  scopes: ["user-top-read", "user-read-recently-played"],
};

function SpotifyTopTracks() {
  const { isLoading, error, data, refresh, retry } = useOAuth(spotifyOAuth);
  const closeWidget = useCloseWidget();
  const [token, setToken] = useState<string | null>(null);
  const [markdown, setMarkdown] = useState("Loading your top tracks...");

  useEffect(() => {
    if (data) setToken(data.accessToken);
  }, [data]);

  useEffect(() => {
    if (!token) return;

    async function fetchTracks() {
      let res = await fetch(
        "https://api.spotify.com/v1/me/top/tracks?limit=10&time_range=short_term",
        { headers: { Authorization: `Bearer ${token}` } },
      );

      if (res.status === 401) {
        const newTokens = await refresh();
        setToken(newTokens.accessToken);
        res = await fetch(
          "https://api.spotify.com/v1/me/top/tracks?limit=10&time_range=short_term",
          { headers: { Authorization: `Bearer ${newTokens.accessToken}` } },
        );
      }

      const body = await res.json();

      if (body.items?.length) {
        const list = body.items
          .map(
            (t: any, i: number) =>
              `${i + 1}. **${t.name}** — ${t.artists.map((a: any) => a.name).join(", ")}`,
          )
          .join("\n");
        setMarkdown(`## Your Top Tracks\n\n${list}`);
      } else {
        setMarkdown("No recent tracks found.");
      }
    }

    fetchTracks().catch((err: Error) => setMarkdown(`Error: ${err.message}`));
  }, [token]);

  if (isLoading) return <Paper markdown="Signing in..." />;

  if (error) {
    return (
      <Paper
        markdown={`**Sign in failed**\n\n${error}`}
        actions={<ActionPanel><Action title="Retry" onAction={retry} style="primary" /></ActionPanel>}
      />
    );
  }

  return (
    <Paper
      markdown={markdown}
      actions={<ActionPanel><Action title="Done" onAction={() => closeWidget("Showed top tracks")} /></ActionPanel>}
    />
  );
}

const SpotifyTopTracksWidget = defineWidget({
  name: "spotify-top-tracks",
  description: "Show your top Spotify tracks",
  schema,
  component: SpotifyTopTracks,
});

export default SpotifyTopTracksWidget;
```
