title: useOAuth
description: Access OAuth tokens and refresh them inside an authenticated widget component

import { Callout } from "fumadocs-ui/components/callout";

Returns the OAuth session that the framework resolved before your component mounted. Use this hook to make authenticated API calls and refresh tokens when they expire.

## Import

Import the hook from the Eney API package:

```typescript
import { useOAuth } from "@eney/api";
```

## Usage

Call the hook at the top of your component to get the current tokens and refresh function:

```typescript
const { accessToken, refreshToken, refresh } = useOAuth();
```

The `accessToken` is always a valid string — the framework guarantees that your component only renders after tokens have been resolved.

## Signature

```typescript
function useOAuth(): OAuthSession
```

### Return value

| Property       | Type                          | Description                                           |
| -------------- | ----------------------------- | ----------------------------------------------------- |
| `accessToken`  | `string`                      | The OAuth access token for making authenticated calls |
| `refreshToken` | `string \| null`              | The refresh token, if the provider returned one       |
| `refresh`      | `() => Promise<OAuthTokens>`  | Exchanges the refresh token for new tokens            |

The `refresh` function:
- POSTs to the provider's token endpoint with `grant_type=refresh_token`
- Returns the new `{ accessToken, refreshToken }`
- Sends the updated tokens to Eney so they persist across sessions
- Throws if no refresh token is available

## Prerequisites

`useOAuth` only works inside widgets that declare an `oauth` configuration in `defineWidget`. If you call it in a widget without OAuth configured, it throws an error.

```typescript
import { defineWidget, useOAuth, Paper } from "@eney/api";
import { z } from "zod";

const schema = z.object({
  query: z.string().optional(),
});

type Props = z.infer<typeof schema>;

const MyWidget = defineWidget({
  name: "my-authenticated-widget",
  description: "Fetches data from an external API",
  schema,
  oauth: {
    clientId: "your-client-id",
    authorizeUrl: "https://provider.com/oauth/authorize",
    tokenUrl: "https://provider.com/oauth/token",
    scopes: ["read", "write"],
  },
  component: MyWidgetComponent,
});
```

## Example

This example fetches data and automatically refreshes the token on a 401 response:

```tsx
import { useState, useEffect } from "react";
import { Paper, useOAuth, useCloseWidget } from "@eney/api";

function MyWidgetComponent(props: Props) {
  const { accessToken, refresh } = useOAuth();
  const closeWidget = useCloseWidget();
  const [token, setToken] = useState(accessToken);
  const [markdown, setMarkdown] = useState("Loading...");

  async function fetchWithRefresh(url: string) {
    let res = await fetch(url, {
      headers: { Authorization: `Bearer ${token}` },
    });

    if (res.status === 401) {
      const newTokens = await refresh();
      setToken(newTokens.accessToken);
      res = await fetch(url, {
        headers: { Authorization: `Bearer ${newTokens.accessToken}` },
      });
    }

    return res;
  }

  useEffect(() => {
    fetchWithRefresh("https://api.example.com/data")
      .then((res) => res.json())
      .then((data) => setMarkdown(JSON.stringify(data, null, 2)))
      .catch((err) => setMarkdown(`Error: ${err.message}`));
  }, []);

  return <Paper markdown={markdown} />;
}
```

<Callout type="info">
  You don't need to handle the sign-in flow yourself. The framework automatically shows a "Signing in..." screen, opens the browser for authorization, and only mounts your component after tokens are available. See the [OAuth guide](/docs/oauth) for the full setup instructions, configuration reference, and how the flow works end to end.
</Callout>

<Callout type="warn">
  Not all OAuth providers issue refresh tokens. If the provider didn't return one, `refreshToken` will be `null` and calling `refresh()` will throw an error. Some providers require specific scopes to get a refresh token (e.g., `offline_access`).
</Callout>
